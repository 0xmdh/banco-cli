# Meeting Pack (Long Form) - Rootstock CLI Agent Proposal

## Objective of This Meeting

Align Rootstock stakeholders around a pilot decision for an Institutional CLI Agent focused on conversion, first funding, and AUM expansion behavior.

## Slide-by-Slide Structure (Suggested)

### 1) Context and Opportunity

- Rootstock institutional thesis is strong.
- Institutional onboarding remains high-friction in practice.
- Opportunity: convert more qualified intent into funded vault activity.

### 2) Core Problem

- Fragmented path from interest to first allocation.
- Too many handoffs across compliance, custody, and execution.
- Limited post-deposit activation loop.

### 3) Proposed Solution

- Rootstock Institutional CLI Agent as operational funnel layer.
- Role-aware, policy-aware guided workflows.
- One system for onboarding + allocation + lifecycle actions.

### 4) What Changes for Rootstock

- Faster movement from lead to funded position.
- Better consistency in institutional onboarding execution.
- Proactive prompts for additional allocation opportunities.

### 5) Demo Story (8-12 min)

- Qualify institutional account.
- Plan first BTC/USD vault allocation.
- Trigger approvals and monitor funded position.
- Show re-deposit recommendation workflow.

### 6) KPI Framework

- Lead -> qualified.
- Qualified -> onboarded.
- Onboarded -> funded.
- First -> second deposit.
- Active institutional AUM influenced by workflow.

### 7) Pilot Proposal (4-6 weeks)

- Discovery and baseline.
- Workflow design and controls.
- Lifecycle activation layer.
- KPI readout and rollout decision.

### 8) Why This Is a Good Fit Now

- Converts existing interest into measurable business outcomes.
- Supports institutional expectations around control and auditability.
- Can launch as phased pilot before deeper integration.

### 9) Decision Request

"Approve a focused pilot with agreed personas, baseline metrics, and success criteria."

## Talk Track (Concise)

### Opening (60 seconds)

"We are not proposing a marketing assistant. We are proposing an institutional growth operator that reduces friction from first conversation to funded vault participation."

### Middle (2-3 minutes)

"The key value is funnel economics: less drop-off, faster first funding, and more repeat allocation behavior. This is measurable and pilotable in weeks, not quarters."

### Close (30 seconds)

"If this direction aligns, we can start with a 4-6 week pilot and return with KPI-backed rollout recommendations."

## Objection Handling Cheatsheet

### "This sounds like just better docs"

"Docs inform; workflows convert. The agent guides and advances state in context."

### "What if deep integration takes too long?"

"Pilot scope is phased. We start with highest-signal journeys and controlled assumptions."

### "What if yield strategy is outside Rootstock rails?"

"This proposal optimizes the capital journey and AUM activity linked to Rootstock participation."

## Meeting Checklist

- Confirm pilot sponsor.
- Confirm pilot personas.
- Confirm baseline data owner.
- Confirm KPI success thresholds.
- Confirm timeline and decision gate dates.
